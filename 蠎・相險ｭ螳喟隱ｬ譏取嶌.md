# 📙 説明書④|広告(Google AdSense)の設定マニュアル

アフィリエイト(成果報酬)に加えて、**表示されるだけで収益になるクリック型広告**を貼る方法です。本命はGoogle AdSenseで、サイト側の対応はすでに組み込み済みです。

## すでに用意してあるもの

| もの | 状態 |
|---|---|
| 広告タグの自動挿入 | Secretを1つ設定するだけで全15ページに入る(自動広告方式) |
| ads.txt | AdSenseに必須のファイル。自動生成されます |
| プライバシーポリシーページ | 審査・規約上の必須要件。`/privacy/` に生成済み(全ページからリンク済み) |
| 広告表記 | 「プロモーションを含みます」表記は掲載済み |

## 設定手順(承認後にやること)

1. **adsense.google.com** でアカウント作成 → サイトURLを登録 → 審査を申請
2. 承認されたら、管理画面に表示される **サイト運営者ID**(`ca-pub-` で始まる数字)をコピー
3. GitHubのSecretに登録:

| Name | Secret |
|---|---|
| `ADSENSE_CLIENT` | `ca-pub-XXXXXXXXXXXXXXXX` |

4. 「Run workflow」で再生成 → 全ページに自動広告が入り、`ads.txt` も自動生成されます
5. AdSense管理画面で「自動広告」をON(広告の量はスライダーで調整。**最初は少なめ推奨**)

## ⚠ 審査について(正直な話)

AdSenseの審査は**自動生成中心のサイトに厳しい**です。「独自性の低いコンテンツ」とみなされると落ちます。通すための現実的な順序:

1. **まずサイトを公開して2〜4週間運用**(更新履歴・トレンドページ・カレンダーが蓄積される)
2. **人間の文章を足す**: トップのフッターやカテゴリページに「このカテゴリの選び方」「予約戦争の傾向」のような運営者の解説文を数百字ずつ追加(generate.pyのフッター文を充実させるか、相談してくれれば解説枠を実装します)
3. 運営者情報(X アカウントへのリンク等)をフッターに書く
4. その状態で審査申請。落ちたら指摘内容に対応して2週間後に再申請(何度でも可)

**審査に通るまでの代替案**: 忍者AdMaxなど審査の緩いクリック広告もありますが、単価がAdSenseの数分の1なので、急がず本命を狙うのがおすすめです。

## 広告とUXのバランス(大事)

- 自動広告の量は**最小〜中**に。広告だらけのサイトは再訪率が落ち、アフィリエイト成約(こちらが本命の収益)も下がります
- このサイトの収益構造は「**アフィリエイト(予約ボタン)が主、AdSenseが従**」が正解です。月間数万PVまでは、AdSense収益は数百〜数千円程度が現実的な期待値です
