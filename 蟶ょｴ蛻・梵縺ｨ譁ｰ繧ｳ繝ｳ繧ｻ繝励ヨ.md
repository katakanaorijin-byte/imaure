# 📊 市場分析と新コンセプト|なぜ「底値レーダー」に作り直したか

## 旧サイト「いまウレ!」の問題点(率直な反省)

楽天の売れ筋ランキングをそのまま表示するサイトは、**楽天自身が公式に表示している情報のコピー**でした。
わざわざ見に来る理由がなく、来た人の購買意欲も不明。「自動で更新される」こと自体には、ユーザーにとっての価値がありませんでした。

## 市場分析(2026年6月時点・出典付き)

### 1. 消費者の最大の関心事は「節約」
- ホットペッパーグルメ外食総研の2026年1月調査:物価高で節約志向が高まった人は**2年連続増加で50.2%と過半数**。食費の節約を意識する人は計63.6%(出典: hotpepper.jp/ggs/research 2026年2月公表)
- 内閣府「消費動向調査」2026年4月:1年後の物価見通しは**「5%以上上昇する」が最多の58.1%**。消費者は「これからも上がる」前提で行動している(出典: esri.cao.go.jp)

### 2. 値上げは「ラッシュ」から「常態化」へ
- 帝国データバンク調査:2025年の食品値上げは**2万609品目**(前年比+64.6%)。2026年はペースが落ち着くが**月1,000品目前後の値上げが常態化**する見通し(出典: 帝国データバンク「食品主要195社」価格改定動向調査)
- → 「値上げのニュース」は飽和。需要は「**じゃあ、どこでいくらで買えば一番安いのか**」という実用情報に移っている

### 3. 象徴的なのが「米」
- 総務省・小売物価統計:米5kgの全国平均は**2026年4月時点で4,562円**。2015年(1,799円)の約2.5倍の水準で高止まり
- 輸入米が3,000円台半ばで売れるなど、消費者は**価格を毎日比較しながら買う行動**が定着している

### 4. 既存サービスの「穴」
- 楽天で「米 10kg」と検索すると、5kg・10kg・27kg・「5kg×2袋」が混在して表示され、**単価での比較ができない**
- 送料の有無でも実質価格が逆転するが、見た目では分からない
- 価格.comは型番のある商品(家電など)が得意で、内容量がバラバラな日用品の単価比較は手薄
- → 「**送料込み・単価換算で生活必需品の本当の最安を毎日計算する**」サービスには明確な空白がある

## 新コンセプト「底値レーダー」

> 楽天の生活必需品を「1kgあたり」「1ロールあたり」の単価に自動換算し、
> 送料無料の商品だけを底値順に毎日並べ直すサイト。
> 毎日の底値を記録し、前日比と推移グラフも表示する。

### 強みの整理

| 観点 | 内容 |
|---|---|
| 独自価値 | 楽天にも価格.comにも無い「日用品の単価正規化」。自分で電卓を叩く手間を代行 |
| 来訪者の質 | 「米を今すぐ安く買いたい」人だけが来る → 購入率(成約率)が構造的に高い |
| 再訪理由 | 価格は毎日動く。前日比・推移グラフは**毎日見る理由**になる |
| 時間の味方 | 価格履歴は日が経つほど蓄積され、サイトの価値が自動で増えていく |
| 検索との相性 | 「米 5kg 最安」「トイレットペーパー コスパ 最強」「米 底値」など、購買直前の検索語と直結 |

### 対象カテゴリ(初期設定)と選定理由

| カテゴリ | 単価 | 選んだ理由 |
|---|---|---|
| お米 | 円/kg | 価格高騰の象徴。検索需要が最大。単価もアフィリエイト額も大きい |
| 水 | 円/L | 重い→ネット購入率が高い。本数×容量で単価が分かりにくい代表格 |
| トイレットペーパー | 円/ロール | 「3倍巻き」問題で比較が最難関 → 換算の価値が最も伝わる |
| ボックスティッシュ | 円/箱 | 同上。まとめ買い需要が大きい |
| 洗濯洗剤(詰め替え) | 円/kg | 大容量詰め替えの単価差が激しく、お得発見が頻発する |
| ドリップコーヒー | 円/杯 | 「1杯あたり◯円」が刺さる嗜好品。リピート購入率が高い |

## 売り方(旧戦略からの変更点)

基本は旧「売れる戦略.md」のSNS運用と同じですが、**投稿の中身が強くなりました**。

- 旧:「今日の売れ筋1位は◯◯」→ ふーん、で終わる
- 新:「**今日の米の底値は598円/kg、前日比▼12円**」→ 数字が具体的で、保存・シェアされやすい。「米の底値を毎朝つぶやくアカウント」はそれ自体がフォローする理由になる

SNS自動投稿(post_sns.py)もこの形式に更新済みです。手動投稿を足すなら「底値が大きく動いた日」「楽天スーパーSALEで底値割れした日」に一言添えるのが最も効きます。

検索(SEO)面では、「(品目名) 最安 単価」「(品目名) 底値」系のキーワードを各カテゴリページが受けられるよう、サイト説明文に品目名を含めています。慣れてきたら「米だけ特化版」を別リポジトリで量産する戦略も有効です(リポジトリをコピーしてCATEGORIESを米数種類にするだけ)。

## 注意点(正直な限界)

- 単価計算は商品名からの自動読み取りのため、**まれに誤差が出ます**(範囲チェックで異常値は自動除外していますが、ゼロにはなりません)。サイト内に「購入前にリンク先で確認」の注意書きを常時表示しています
- 収益は引き続きアクセス数次第です。ただし旧版と違い「来た人が買う確率」と「毎日来る理由」が構造的に改善されています

# 追記:saiyasune.com(最安値.com)との比較と「上位互換」の考え方

| | 最安値.com | 底値レーダー |
|---|---|---|
| 対応モール | Amazon・楽天・Yahoo!ほか多数 | 楽天・Yahoo!(公式API・規約準拠) |
| バーコード検索 | ○ | ✕(静的サイトでは不可) |
| ポイント計算 | 詳細なシミュレーター | 簡易版(自分の還元率%を入力→実質価格順) |
| **単価換算(円/kg等)** | ✕ | **○(最大の差別化)** |
| **底値の毎日記録・推移グラフ** | ✕ | **○** |
| **送料込みだけの公平比較** | △ | **○(初期設定で強制)** |
| 運用コスト | サーバー運用(アクセス制限あり) | **0円・完全自動**(GitHub Pages) |

結論:**全機能での上位互換は狙わない**。最安値.comは「型番のある商品(家電など)をバーコードで探す」用途に強い。
底値レーダーは「**内容量がバラバラな生活必需品を、単価で毎日比較する**」という、向こうがカバーしていない軸で上位を取る。
土俵を変えて勝つのが、無料・自動運用の個人サイトの正しい戦い方です。

なおAmazonを入れていないのは技術的怠慢ではなく規約上の制約です(商品APIの利用にはアフィリエイト売上3件以上が必要・スクレイピングは禁止)。条件を満たした時点で追加できます。

# 追記2:最終コンセプト「予約開始レーダー」への転換理由

価格比較(底値)案はユーザー判断により見送り。「最安値以外」の軸として、**自動監視そのものが商品になる**コンセプトに転換した。

**選定理由**
1. **需要の質**: ポケモンカード・人気フィギュア・ガンプラは予約開始から数時間〜数日で受付終了が常態化しており、「予約開始をいち早く知る」こと自体に強い需要がある。知った人はその場で予約する(=成約率が構造的に高い)
2. **自動化との相性**: 「毎日全カテゴリを巡回して昨日との差分を取る」のは人間には不可能で機械には簡単。AIサイトの弱点(人間が書く記事に勝てない)ではなく強み(休まない監視)で勝負できる
3. **報酬単価**: フィギュア1〜3万円・カードBOX5千〜2万円・ゲーム1万円と高単価。報酬率2〜4%なら1成約数百円〜千円超
4. **SNS適性**: 「予約開始」情報は保存・共有・通知オンの動機になる。新着0件の日は投稿を自動スキップしてノイズを出さない設計

**仕組み**: 楽天APIを毎日「新着順」で巡回 → 商品コードを `seen.json` に記録 → 記録に無いコード = 予約開始したばかりの商品として「NEW」表示+SNS速報。

**限界(正直に)**: 1日1回更新のため「分単位の速報」ではない(更新頻度はcron変更で1日数回まで上げられる)。再販・新店舗の取扱開始もNEWに含まれる(サイト内に明記済み)。アクセスを集める努力が必要なのは他案と同じだが、「◯◯(作品名) 予約」系の検索流入と、予約情報系SNSアカウントの定番フォーマットという確立した集客経路がある。
