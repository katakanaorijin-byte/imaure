# 📕 説明書|「予約開始レーダー」公開マニュアル(完全初心者向け)

このサイトは **お金が一切かからず**、一度設定すれば **15分ごとに自動で更新** されます。
パソコンに何かをインストールする必要もありません。**すべてブラウザだけで完結**します。

作業時間の目安:**合計 約40分**(すべて初回だけ。2日目以降は何もしなくてOK)

---

## 全体の流れ(3ステップ)

| ステップ | やること | 時間 |
|---|---|---|
| ① | 楽天アフィリエイトに登録して「2つのID」をもらう | 約15分 |
| ② | GitHub(無料サービス)にサイトを置く | 約15分 |
| ③ | 自動更新のスイッチを入れて公開する | 約10分 |

---

## ステップ① 楽天アフィリエイトに登録する

### 1-1. 楽天アフィリエイトに登録(無料・審査なし)

1. ブラウザで「**楽天アフィリエイト**」と検索して公式サイトを開く(affiliate.rakuten.co.jp)
2. 楽天会員のIDでログインする(楽天会員でなければ先に無料登録)
3. 利用規約に同意して登録完了

### 1-2. アフィリエイトIDをメモする

1. 楽天アフィリエイトにログインした状態でトップページを開く
2. ページ内の「**基本情報**」や「**アフィリエイトID**」と書かれた場所を探す
3. `1234abcd.5678efgh.1234abcd.5678efgh` のような英数字の列が **アフィリエイトID** です
4. メモ帳などにコピーして保存(**ID その1**)

### 1-3. 楽天APIのキー(アプリID)をもらう

「サイトが毎日自動でランキングを取りに行くための鍵」です。これも無料です。

1. 「**楽天ウェブサービス**」と検索して公式サイトを開く(webservice.rakuten.co.jp)
2. 楽天IDでログイン →「**アプリID発行**」をクリック
3. フォームに入力する:
   - アプリ名: `予約開始レーダー`(なんでもOK)
   - アプリURL: `https://example.com`(あとで本物に変えればOK)
4. 発行された **アプリID(applicationId)** という長い数字をメモ(**ID その2**)

> ✅ ここまでで「アフィリエイトID」と「アプリID」の2つがメモできていれば成功です。

## ステップ② GitHubにサイトを置く

GitHub(ギットハブ)は、ファイルを置いておくと**無料でホームページとして公開してくれる**サービスです。世界中で使われている安全なサービスです。

### 2-1. GitHubアカウントを作る

1. github.com を開いて「**Sign up**」をクリック
2. メールアドレス・パスワード・好きなユーザー名を入力して登録
   - ⚠ ここで決めた**ユーザー名がサイトのURLの一部**になります(例: ユーザー名が `tanaka123` ならサイトは `tanaka123.github.io/...`)

### 2-2. リポジトリ(サイトの置き場所)を作る

1. ログイン後、右上の「**+**」→「**New repository**」をクリック
2. 次のように入力:
   - Repository name: `imaure`(これもURLの一部になります)
   - **Public** を選ぶ(Publicでないと無料公開できません)
3. 「**Create repository**」をクリック

### 2-3. ファイルをアップロードする

このフォルダに入っている次のファイルを、**フォルダ構成そのまま**アップロードします。

```
imaure/
├── generate.py                  ← サイトを作るプログラム
├── docs/
│   └── index.html               ← サイト本体(自動で作り直されます)
└── .github/
    └── workflows/
        └── update.yml           ← 毎日自動更新の設定
```

アップロード手順:

1. 作ったリポジトリのページで「**uploading an existing file**」というリンクをクリック
2. パソコンの `imaure` フォルダの**中身を全部**ドラッグ&ドロップ
3. 下の「**Commit changes**」ボタンをクリック

> ⚠ もし `.github` フォルダがドラッグでうまく上がらない場合:
> リポジトリページの「Add file」→「Create new file」を選び、ファイル名の欄に
> `.github/workflows/update.yml` とそのまま入力 →
> パソコン上の `update.yml` の中身をコピーして貼り付け → Commit changes。
> 同じやり方で `generate.py` と `docs/index.html` も作れます。

## ステップ③ 自動更新のスイッチを入れて公開する

### 3-1. 2つのIDをGitHubに登録する(シークレット設定)

ステップ①でメモした2つのIDを、安全な金庫(Secrets)に入れます。

1. リポジトリページ上部の「**Settings**」をクリック
2. 左メニューの「**Secrets and variables**」→「**Actions**」をクリック
3. 「**New repository secret**」をクリックして1つ目を登録:
   - Name: `RAKUTEN_APP_ID`(**この通り正確に**。大文字・アンダーバー含む)
   - Secret: メモした**アプリID**(数字の長い列)を貼り付け
   - 「Add secret」をクリック
4. もう一度「New repository secret」で2つ目を登録:
   - Name: `RAKUTEN_AFFILIATE_ID`
   - Secret: メモした**アフィリエイトID**を貼り付け

### 3-2. サイトを公開する(GitHub Pages)

1. 同じ「Settings」の左メニューから「**Pages**」をクリック
2. 「Build and deployment」の設定:
   - Source: **Deploy from a branch**
   - Branch: **main** を選び、フォルダは **/docs** を選ぶ
3. 「**Save**」をクリック
4. 1〜2分待ってページを再読み込みすると、上部に
   `Your site is live at https://あなたのユーザー名.github.io/imaure/`
   と表示されます。**これがあなたのサイトのURLです!**

### 3-3. 最初の更新を手動で実行する(本物のランキングに切り替え)

1. リポジトリページ上部の「**Actions**」タブをクリック
2. (初回は「I understand... enable them」のようなボタンが出たらクリックして有効化)
3. 左の「**サイトを毎日自動更新**」をクリック
4. 右側の「**Run workflow**」→ 緑の「**Run workflow**」ボタンをクリック
5. 1〜2分で緑のチェック✅がつけば成功。サイトを開くと**本物の楽天ランキング**に変わっています
   (変わらないときはブラウザを強制再読み込み:スマホは引っ張って更新、PCは Ctrl+Shift+R)

> 🎉 **これで完成です。** 明日からは15分ごとに、何もしなくても自動で最新情報に更新されます。

### 3-4. 楽天アフィリエイトにサイトURLを登録する(重要)

楽天アフィリエイトの管理画面の「サイト情報の登録」に、できあがったURL
(`https://あなたのユーザー名.github.io/imaure/`)を登録してください。
**未登録のサイトで成果が発生すると報酬が支払われないことがあります。**

## 💰 報酬の受け取りについて

- 誰かがあなたのサイトのリンクから楽天で買い物をすると、**商品代金の2〜4%程度**(ジャンルによる)が紹介料になります
- 報酬は**楽天キャッシュ(楽天ポイントとして使える電子マネー)**で支払われます。月3,000円を超える分を銀行振込にするには楽天銀行口座などの条件があります(詳細は楽天アフィリエイトのヘルプ参照)
- リンクを踏んだ人が**その後30日以内に買った別の商品**でも成果になります(カートに入れてから89日以内の購入)

## 🔧 よくあるカスタマイズ

すべて `generate.py` の上の方にある「設定」部分を書き換えるだけです。
GitHub上で `generate.py` を開いて鉛筆マーク(Edit)→ 書き換え → Commit changes で反映されます。

**サイト名を変えたい**
```python
SITE_NAME = "底値レーダー"   ← ここを好きな名前に
```

**カテゴリ(監視ジャンル)を変えたい・増やしたい**
`CATEGORIES` のリストに項目を足したり消したりします。1カテゴリはこういう形です:
```python
{"name": "ポケモンカード", "icon": "🃏", "keyword": "ポケモンカード 予約 BOX",
 "ng": "中古 オリパ スリーブ", "min_price": 3000},
```
`keyword` は楽天で検索する言葉(「予約」を含めるのがコツ)、`ng` は除外したい言葉、`min_price` はこれ未満の安い雑貨を弾く下限価格です。
例えば「鉄道模型」を監視したいなら `{"name": "鉄道模型", "icon": "🚃", "keyword": "鉄道模型 予約", "ng": "中古 ジャンク", "min_price": 3000}` を足すだけです。

**表示する商品数を変えたい**
```python
ITEMS_PER_CATEGORY = 14
```

**新着判定の記録(seen.json)について**
一度レーダーに映った商品は `docs/seen.json` に記録され、翌日からの「NEW」判定に使われます。**このファイルは消さないでください**(消すと翌日の表示が全部NEWになってしまいます)。なお運用初日はすべての商品が初対面なので、NEW判定は**2日目から**始まります。

**更新時刻を変えたい**
`.github/workflows/update.yml` の `cron: "0 21 * * *"` を変更。
時刻は**世界標準時(日本時間マイナス9時間)**で書きます。例:日本時間の昼12時 → `"0 3 * * *"`

## ❓ トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| サイトが「404 Not Found」 | Pages設定のフォルダが `/docs` になっているか確認。公開直後は数分かかることもある |
| ずっと「デモデータ表示中」のまま | Secretsの名前が `RAKUTEN_APP_ID` と1文字でも違うと読み込まれません。スペル確認 → Actionsから再実行 |
| Actionsが赤い❌になる | ❌をクリック→「サイトを生成」を開くとエラー内容が見えます。「警告」だけなら一時的なAPI不調の可能性が高いので、再実行してみてください |
| 商品画像が出ない | 楽天側で画像未登録の商品です。異常ではありません |
| 報酬が発生しない | サイトURLを楽天アフィリエイトに登録したか確認。あとはアクセスを増やすのが本題です(→「売れる戦略」参照) |

## ⚠ 守ること(大事)

1. **広告であることの表示を消さない** … サイト内の「本ページはプロモーションを含みます」の表記は、法律(景品表示法のステマ規制・2023年10月施行)で必要です。`generate.py` から消さないでください
2. **楽天の規約を守る** … 価格を自分で書き換える、誇大な表現を足す、などはNG
3. **API の呼びすぎ注意** … 1日1回の自動更新なら全く問題ありません
4. **収益は保証されません** … この仕組みは「サイトを作って維持する手間」をゼロにするものです。稼げるかどうかはアクセス数で決まります(増やし方は「売れる戦略.md」へ)
