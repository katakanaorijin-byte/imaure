# 📗 説明書③|Yahoo!連携・横断検索・ポイント設定マニュアル

このバージョンで増えた機能と、その設定方法です。

| 機能 | 設定しないと | 設定すると |
|---|---|---|
| Yahoo!ショッピング連携 | 楽天だけで動く(問題なし) | 毎日の底値リストと検索が**2モール横断**になる |
| 横断検索バー | キー未設定だと検知済み商品のみ検索 | 検知済み商品+Yahoo!の**全件検索**ができる |
| 中継API(Cloudflare Workers) | 楽天は検知済み商品のみ検索 | 楽天市場・Yahoo!ショッピングの**全商品**を横断検索バーから検索できる |
| Yahoo!リンクの収益化 | Yahoo!リンクは報酬なし | ValueCommerce経由で**Yahoo!でも紹介料**が入る |

> ℹ️ 楽天の現行APIは`accessKey`(秘密情報)が必須のため、ブラウザから楽天の全商品を直接検索することはできません。横断検索バーの「楽天」は標準では**サイトが検知済みの商品の中だけ**を探し、それ以外は「楽天市場で検索」リンクに案内します。`SEARCH_API_URL`(パートA')を設定すると、楽天もYahoo!も全商品を横断検索バーから検索できるようになります。

どれも後回しにできます。**楽天だけでもサイトは完全に動きます。**

## パートA:Yahoo!ショッピング連携(無料・約10分)

Yahoo!のAPIキー(Client ID)を1つ取るだけです。

1. **Yahoo!デベロッパーネットワーク**(developer.yahoo.co.jp)を開き、Yahoo! JAPAN IDでログイン(なければ無料登録)
2. 「**アプリケーションの管理**」→「**新しいアプリケーションを開発**」をクリック
3. フォーム入力:
   - アプリケーションの種類: **クライアントサイド**
   - アプリケーション名: `底値レーダー`(なんでもOK)
   - サイトURL: あなたのサイトURL
4. 作成すると **Client ID** という長い文字列が表示されるのでコピー
5. GitHubのリポジトリ →「Settings」→「Secrets and variables」→「Actions」で登録:

| Name(この通り正確に) | Secret |
|---|---|
| `YAHOO_APP_ID` | コピーしたClient ID |

6. 「Actions」タブから「Run workflow」で手動実行すると、底値リストに紫色の「Yahoo!」バッジ付き商品が混ざり始めます

> 💡 **キーがページに表示される件について**
> 横断検索はブラウザから直接APIを呼ぶため、Yahoo Client IDはサイトのソースに埋め込まれます。これは**公開して使う前提の無料キー**(支払い情報とは無関係)で、Webサイトに埋め込むのはYahoo!が想定している通常の使い方です。楽天アフィリエイトIDも、リンクに必ず含まれる公開情報なので問題ありません(楽天の`accessKey`だけは秘密情報のため、サーバー側=GitHub Actionsだけで使い、ブラウザには出しません)。

## パートA':楽天市場・Yahoo!の全商品を横断検索バーで検索できるようにする(Cloudflare Workers・無料・約20分)

`RAKUTEN_APP_ID`/`RAKUTEN_ACCESS_KEY`(説明書①参照)は**毎日の自動巡回専用**です。楽天の現行APIは`applicationId`と`accessKey`をセットで要求する仕様のため、`accessKey`を安全に置けないブラウザから楽天の商品検索APIを直接叩くことができません(`accessKey`をページに埋め込んだ瞬間、誰でも見えてしまうため)。

これを解決するため、`accessKey`を安全に保管できる小さな中継サーバー(**Cloudflare Workers**、無料枠で十分動きます)を1つ用意します。設定しなくてもサイトは問題なく動きます(楽天は検知済み商品のみの検索になります)。

### A'-1. Cloudflare Workersをデプロイする

このフォルダの `worker/` ディレクトリに中継API一式が入っています。

1. **Cloudflareアカウント**(dash.cloudflare.com)を無料登録
2. パソコンに **Node.js** をインストール済みなら、`worker/` フォルダで以下を実行:
   ```
   cd worker
   npm install
   npx wrangler login        ← ブラウザが開くのでCloudflareでログイン
   npx wrangler deploy       ← デプロイ実行
   ```
3. デプロイが終わると `https://yoyaku-radar-search.あなたのサブドメイン.workers.dev` のようなURLが表示されます。これがあなたの**中継APIのURL**です(末尾に何も付けない状態でメモ)

### A'-2. Cloudflare Workersに秘密のキーを設定する

`accessKey`はここにだけ保存します(GitHub・ブラウザには出ません)。`worker/` フォルダで:

```
npx wrangler secret put RAKUTEN_APP_ID
npx wrangler secret put RAKUTEN_ACCESS_KEY
npx wrangler secret put RAKUTEN_AFFILIATE_ID
npx wrangler secret put YAHOO_APP_ID
```

それぞれ実行すると値の入力を求められるので、説明書①・パートAでメモした値を貼り付けます(`RAKUTEN_AFFILIATE_ID`・`YAHOO_APP_ID`は持っていれば)。

### A'-3. GitHubに中継APIのURLを登録する

1. GitHubのリポジトリ →「Settings」→「Secrets and variables」→「Actions」
2. 「New repository secret」で登録:

| Name(この通り正確に) | Secret |
|---|---|
| `SEARCH_API_URL` | A'-1でメモした中継APIのURL + `/search`(例: `https://yoyaku-radar-search.xxx.workers.dev/search`) |

3. 「Actions」タブから「Run workflow」で再生成すると、横断検索バーの「楽天(検知済み商品)」表記が「楽天」に変わり、商品名で検索すると**楽天市場・Yahoo!ショッピングの全商品**がヒットするようになります

> 💡 ローカルで検索バーの見た目だけ確認・修正したい場合は、`SEARCH_API_URL` を環境変数に入れて `python sync_search_widget.py` を実行すると、APIキー不要でこのURLだけをページに反映できます。

## パートA'':ローカルでsearch_assets.pyを編集したときの反映方法

`search_assets.py`(検索バーの見た目や動き)を直しても、`docs/*/index.html` は自動では更新されません。本来は `python generate.py` を実行すれば反映されますが、手元のパソコンに楽天/YahooのAPIキーが無い場合、`generate.py` を実行すると**本番の商品データがデモデータに置き換わってしまいます**。

そのため、検索バーの見た目・動きだけを直したいときは、代わりに次のコマンドを使ってください(APIキー不要・商品データには触りません):

```
python sync_search_widget.py
```

これは `docs/` 配下の生成済みHTMLを開いて、検索バーのCSS/HTML/JS部分だけを `search_assets.py` の最新内容に差し替えます。各ページに既に埋め込まれているAPIキー設定は維持されます。GitHub Actions側は毎回 `generate.py` をフルで実行して同じ内容を作るため、本番では特別な対応は不要です。

## パートB:Yahoo!リンクの収益化(ValueCommerce・約20分+審査待ち)

楽天は直接アフィリエイトできますが、Yahoo!ショッピングの紹介料は「**バリューコマース**」というASP(広告仲介会社)経由が定番です。**LinkSwitch**という仕組みを使うと、サイト内のYahoo!リンクが**自動で全部、報酬リンクに変換**されます(1リンクずつ作業する必要なし)。

1. **バリューコマース**(valuecommerce.ne.jp)で無料のアフィリエイト会員登録(サイトURLを登録 → 審査あり・通常数日)
2. 承認されたら管理画面で「**Yahoo!ショッピング**」のプログラムを検索して提携申請(即時〜数日)
3. 管理画面の「ツール」→「**LinkSwitch**」→「LinkSwitchを利用開始する」
4. 表示されるタグの中に `vc_pid = "888888888"` のような**数字(あなたのID)**があるのでコピー
5. GitHubのSecretに登録:

| Name | Secret |
|---|---|
| `VC_PID` | コピーした数字 |

6. 次回の自動更新から、ページにLinkSwitchが組み込まれ、Yahoo!リンク経由の購入で報酬(商品代金の1%程度〜)が発生するようになります

> ⚠ 審査に落ちた場合は、サイトに記事(「市場分析」の内容を運営者コメントとして載せる等)を少し足してから再申請すると通りやすくなります。

## パートC:横断検索バーの使い方(設定不要)

パートAまで終わっていれば、サイト上部の検索バーがそのまま使えます。

- **送料無料のみ**(初期ON):送料の罠を避けて公平に比較。OFFにすると候補は増えますが「送料別の場合あり」表示が付きます
- **並び替え**:
  - 価格が安い順(ふつうの最安)
  - **単価が安い順**:商品名から内容量を自動判定して「約◯円/kg」チップを表示し、単価で比較(このサイト最大の武器)
  - **ポイント込み実質順**:↓のポイント設定を引いた実質価格で比較
- **ポイント還元率を設定**:「楽天3%・PayPay5%」のように自分の還元率を入れると保存され、実質価格の計算に使われます(saiyasuneの「ポイントシミュレーター」の簡易版です)
- 検索すると、入力欄の下に結果パネルが**浮いて表示**され、その下のタブ・カテゴリ一覧は一時的に隠れます(ページが続けて伸びて見える問題への対策)。「ホームに戻る」を押すと元の表示に戻ります

## ❓ トラブルシューティング

| 症状 | 対処 |
|---|---|
| 検索バーで楽天が「検知済み商品」しか出ない | 仕様です。`SEARCH_API_URL`(パートA')を設定すると楽天市場の全商品も検索できるようになります |
| 検索バーの「Yahoo!」チェックがグレーで押せない | `YAHOO_APP_ID`・`SEARCH_API_URL`のどちらも未設定です(意図した挙動)。パートA/A'の手順で設定して再生成すると有効になります |
| 検索で「検索API」または「Yahoo!は取得できませんでした」(チェックは押せる状態) | `SEARCH_API_URL`/Client IDの貼り間違いか、一時的なAPI不調です。Secretを確認 → サイト再生成。Workerの場合はCloudflareダッシュボードのログも確認できます |
| Yahoo!の商品が底値リストに出ない | そのカテゴリで「送料無料」の商品が見つからなかっただけの場合があります(異常ではない) |
| 検索結果が0件 | 「米10kg」→「米 10kg」のように**スペース区切り**にすると改善します |
| Amazonは? | 下の「パートF」を参照。段階的に対応できます |

## パートD:検索の予測変換(サジェスト)について

検索バーに文字を入れると、候補が自動で表示されます。設定は不要です。

- **ホビー用語辞書**(作品名・シリーズ名など約110語を内蔵)と、**あなたの検索履歴**(最新10件・ブラウザに保存)から候補を出します
- ひらがな・カタカナどちらで打っても一致します(「ぽけもん」→「ポケモンカード」)
- ↑↓キーで選択、Enterで検索、Escで閉じる、にも対応しています
- 外部サービスを使わない自前方式なので、高速で、サービス終了の影響も受けません

**候補の言葉を追加・変更したいとき**は、`search_assets.py` の中の `var DICT = [...]` のリストに言葉を足すだけです(GitHub上で編集 → Commit changes → 次回更新で反映)。流行りの作品名を足しておくと検索が便利になります。

## パートE:監視カテゴリ(12種類)

現在の監視対象: ポケモンカード/ワンピースカード/遊戯王・他カード/フィギュア/ねんどろいど・figma/ガンプラ・プラモデル/ゲームソフト/アニメBlu-ray・CD/特撮・なりきり玩具/鉄道模型/トミカ・ミニカー/ぬいぐるみ・グッズ

各カテゴリは1つの`keyword`だけでなく、`keywords`(複数形)に追加の検索語をいくつでも登録できます(例: 「ねんどろいど・figma」は`ねんどろいど`に加えて`figma`/`ポップアップパレード`/`ルックアップ`も検索)。カテゴリ名が示す範囲を実際に検索できているか、ときどき見直すのがおすすめです。

カテゴリの増減は説明書①のカスタマイズ章を参照してください(1カテゴリ=設定3行)。カテゴリ・検索語を増やすと毎日の取得時間が1つあたり約1.2秒ずつ延びますが、現在の46キーワード程度であれば全く問題ありません。

## パートF:Amazon対応(段階制・約15分)

Amazonの商品データAPI(PA-API)は「**アソシエイト登録+180日以内に売上3件**」が利用条件のため、価格比較への完全組み込みは最初からはできません(スクレイピングは規約違反のため行いません)。
代わりにこのサイトは**「Amazonで探す」リンク方式**に対応しています。これはAPI不要・規約OKで、売上実績を貯める入口にもなります。

### F-1. Amazonアソシエイトに登録する

1. **affiliate.amazon.co.jp** を開き、Amazonアカウントでログインして登録
2. サイト情報にあなたのサイトURL(`https://〜.github.io/imaure/`)を登録
3. 登録完了画面やアカウント設定に表示される「**ストアID/トラッキングID**」をコピー
   (`xxxxx-22` のような形式。これがあなたのタグです)

### F-2. GitHubにタグを登録する

| Name(この通り正確に) | Secret |
|---|---|
| `AMAZON_PARTNER_TAG` | コピーしたトラッキングID(例: `xxxxx-22`) |

登録して「Run workflow」すると、各商品カードに白い「**Amazonで探す**」ボタンが、横断検索の結果にも「**Amazonでも検索 →**」リンクが付きます(商品名から【予約】などの飾りを除いた検索リンクで、あなたのタグ入り)。

### F-3. 仕組みと今後のロードマップ

- リンク経由でAmazonで何か売れると報酬が入り、同時に**本審査+PA-API解禁の条件である「売上3件」**にカウントされます
- ⚠ アソシエイトは登録から**180日以内に3件の売上**がないとアカウントが閉鎖されます(再登録は可能)。サイトのアクセスが育ってから登録するのも手です
- **売上3件を達成してPA-APIが使えるようになったら**、楽天・Yahoo!と同じようにAmazonの価格・在庫をサイトに直接表示する本格対応が可能になります。そのときはこのZIPの構成のまま `fetch_amazon` を追加する改造になります

あみあみ・駿河屋などホビー専門店の多くは、バリューコマースやA8.netに広告プログラムがあります。パートBで作ったバリューコマースのアカウントで提携すれば、「◯◯(専門店)で探す」リンクを同じ方式で足せます。
